# Contributing

Thanks for considering a contribution. Skills compound when other operators sharpen them. Here's how to help.

## What's most welcome

- **Bug fixes** in skill prompts, scripts, or templates
- **Real-world feedback** — "I used this in production and X broke." Open an issue.
- **New reference docs** that codify a pattern you've battle-tested
- **Improvements to existing references** — tighter examples, current API specs, fewer assumptions
- **Cross-skill links** — if you spot a missing pointer between this skill and another in the stack, PR it in

## What's less welcome

- **Cosmetic refactors** with no functional change
- **Renaming things** unless the new name is genuinely clearer
- **Mass formatting changes** that mix substantive edits with whitespace
- **AI-generated prose** that reads like AI-generated prose (use the [voice-profile-kit](https://github.com/nickyc1/voice-profile-kit) to keep your contributions human)

## Process

1. Fork the repo
2. Make your change on a branch
3. Open a PR with:
   - One-line summary of what you changed
   - One-line summary of why
   - If you're adding a new reference, link it from the SKILL.md
4. I'll review within a week

## Style notes

- Keep SKILL.md and README.md readable for an operator scanning it for the first time
- Reference subdocs can go deeper, but earn their length — no padding
- No em dashes for dramatic effect, no "It's not X, it's Y" framing
- Real examples beat abstract explanations
- Specifics beat generics
